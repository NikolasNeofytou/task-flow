## üìù Documentation Guidelines

### When to Create New Documentation

Create a new document when:
- Adding a major feature with complex architecture
- Documenting a significant system or pattern
- Completing a development phase
- Creating reference material for multiple developers

### When to Update Existing Documentation

Update existing docs when:
- Fixing errors or outdated information
- Adding clarifications or examples
- Expanding on existing topics
- Updating version numbers or dependencies

### Documentation Standards

- **Clear Structure**: Use headings, lists, and tables
- **Code Examples**: Include practical code snippets
- **Visual Aids**: Add diagrams, screenshots when helpful
- **Links**: Cross-reference related documents
- **Maintenance**: Keep version and date updated
